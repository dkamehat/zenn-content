## 何を作ったか

キーを押して話すと、整形済みのテキストが「いま開いているアプリ」に入力される——
そういう音声入力ツール **Koe（声）** を作って、OSSで公開しました（MIT）。

特徴は一言で **「100%ローカル・オフライン」**。文字起こしも文章整形も全部自分のPCの
GPUで動き、音声もクラウドに一切出ません。

- リポジトリ: https://github.com/dkamehat/Koe
- Windows用。スタンドアロンの実行ファイル（Python不要）か、ソースから実行。

![Koe demo](https://github.com/dkamehat/Koe/raw/main/docs/demo.gif)

## なぜ作ったか

着想は [Aqua Voice](https://withaqua.com/) です。「押して話すだけで整った文章が入る」
体験は最高なんですが、私は2つ条件がありました。

1. **サブスクではなく、ゼロ円で使いたい**
2. **音声や画面の内容を、マシンの外に出したくない**（仕事上ここが固い要件）

この2つを満たすには「ローカルで動く」しかない。そこで自分のテーゼを
**「ローカルで動くことは必須 / 誰でも使えて、誰でも安全」** に置いて作りました。

## 構成

```
 マイク ─► 録音 ─► faster-whisper ─► 辞書 ─► ③整形 ─► クリップボード貼付 ─► アプリ
            ①        ②(CUDA)       固有名詞   ローカルLLM    Unicode安全        ④
                                            (文脈反映・ストリーミング)
```

- **② STT**: faster-whisper（CTranslate2）を CUDA / float16 で。既定は large-v3-turbo。
- **③ 整形**: ローカルLLM（Ollama の qwen2.5）でフィラー除去・句読点付与・固有名詞の表記揺れ補正。
  **原文の言い回しは保持／翻訳は絶対しない**。クラウド（Claude/OpenAI）も使えるが opt-in・
  キーは環境変数のみ・選ぶと画面文脈の読み取りは自動でオフ。

## ハマったところ（知見）

ここが本題。動くものを作るより、「速くて気持ちいい」にするまでが大変でした。

### 1. `localhost` で毎リクエスト +2.2秒（Windows）

体感で出力まで3〜5秒かかり、最初はモデルが遅いのかと疑いました。が、計測すると
推論は速い。犯人は **Python の `requests` が `http://localhost:11434` を解決する際の
IPv6→IPv4 フォールバックで、1リクエストあたり約2.2秒**。`127.0.0.1` に変え、
`Session(trust_env=False)` を使い回したら、整形が ~2.6秒 → ~0.3〜0.5秒に。
"遅延問題"の9割はこれでした。Windowsでローカルサービスを叩くなら `localhost` を使わない。

### 2. Whisper の「繰り返し幻聴」

無音やノイズで、Whisper が `シャッシャッシャッ…` のようにトークンをループすることがある。
`compression_ratio_threshold` 等の標準ガードを入れても稀に貫通する。しかも、その巨大な
繰り返し文字列を③のLLMが真面目に処理して10秒浪費する二次被害つき。最終的に
**「同じ短い単位が6回以上連続したら1個に畳む」確定的な後処理ガード**を足して解決しました。

### 3. 頭欠け（「…ello」問題）をプリロールで

キーを押した瞬間はマイクのドライバ起動レイテンシで最初の一音が欠ける。そこで
**常時オンのリングバッファ**を回し、録音開始時に直前0.3秒を前置することで頭欠けを解消。

### 4. ③ローカルLLMの暴走を抑える

ローカルLLMは放っておくと「翻訳する」「言い換える」をやりがち。日本語が英語になって
出てきた時は驚きました。プロンプト強化に加え、**入力にCJKがあるのに出力からCJKが
消えていたらLLM結果を破棄する**決定的な言語ガードを実装。"賢さ"より"予測可能性"を優先。

### 5. CUDA同梱のexe化

「誰でも使える」を満たすには Python/git なしで動く配布物が要る。PyInstaller で
スタンドアロン化したものの、CUDA DLL（cuBLAS/cuDNN）が曲者。CTranslate2 のローダーは
**自分のパッケージ位置とバンドルのルートしか探索しない**ので、DLLはルートに置く必要がある。
さらに PyInstaller のフックが同じDLLを別フォルダにも二重収集していて、それを削って
~650MB 削減。最終的に 1.2GB の zip で配布できました。

## 使い方

```powershell
# A) ダウンロード（Python不要）
#    Releases から Koe-win64-cuda.zip を解凍 → Koe.exe をダブルクリック

# B) ソースから
git clone https://github.com/dkamehat/Koe.git
cd Koe
.\setup.ps1
.\run-admin.bat
```

Right Ctrl を1回押す → 話す（途中で止まってOK）→ もう1回押す → 整形済みテキストが入ります。
